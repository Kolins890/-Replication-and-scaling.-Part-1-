Домашнее задание к занятию «Репликация и масштабирование. Часть 1»

Задание 1

На лекции рассматривались режимы репликации master-slave, master-master, опишите их различия.

Master‑slave репликация:

Направление данных: однонаправленное — от master к slave.

Запись: возможна только на master‑сервере. Slave‑серверы работают в режиме «только для чтения» (read‑only).

Назначение: масштабирование чтения (запросы SELECT можно распределять на slave‑серверы), резервное копирование, аналитика без нагрузки на основной сервер.

Отказоустойчивость: ограничена — при падении master‑сервера требуется ручное переключение на slave (или автоматизация через сторонние инструменты).

Конфликты: отсутствуют, так как запись идёт только в одном месте.

Простота настройки: относительно простая конфигурация.

Master‑master репликация:

Направление данных: двунаправленное — каждый сервер может быть и master, и slave одновременно.

Запись: возможна на любом из серверов. Изменения реплицируются на все узлы.

Назначение: высокая доступность (HA), балансировка нагрузки по записи, географическое распределение данных.

Отказоустойчивость: выше — при падении одного сервера другой продолжает работать и принимать записи.

Конфликты: возможны (например, одновременное изменение одной строки на разных серверах). Требуются механизмы разрешения конфликтов (UUID, автоинкремент с разными шагами и т. д.).

Сложность настройки: более сложная конфигурация и поддержка.
